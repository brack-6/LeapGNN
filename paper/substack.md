# The Machine Found a Physics Trick

*Mantecanaut Dispatch*

---

Every morning at 04:00, a script on my server crosses GitHub repositories against arXiv papers and looks for surprising semantic overlap. It calls these collisions. Most are noise. Some are interesting. Occasionally one is genuinely strange.

Last week it collided a paper about why LLM leaderboards are misleading with a TypeScript AI assistant called OwnPilot. The distance score was 0.496 — close enough to be suspicious, far enough to be surprising. The report it generated said both systems were doing the same thing underneath: building sparse bipartite preference matrices and factorising them into low-rank embeddings.

I greenlit it. A build worker scaffolded a TypeScript file. I mostly ignored it.

Then I actually read the collision report.

The isomorphism it found was real: both systems ask "who should win a duel?" — LLM vs human evaluator in one case, AI agent vs tool in the other. The same tournament structure. The same low-rank factorisation.

Following that thread led somewhere unexpected: if you treat a graph as a physical system — atoms as oscillators, bonds as springs — message passing becomes physics. Instead of averaging your neighbours' features, you run the leapfrog integrator. You conserve energy. The transformation becomes reversible.

Standard GNNs lose information at every layer. This one doesn't.

I tested it on MUTAG, a benchmark dataset of 188 molecules labelled by mutagenicity. The physics GNN hit 92.6% accuracy averaged across 10 random seeds. Standard GCN: 83.2%. A 9-point gap on the first run of a prototype generated by a collision detection script.

On PROTEINS — a harder dataset with fewer node features — the gap disappears entirely. The physics prior only works when there's actual physics to exploit. Molecules have it. Protein graphs, apparently, don't.

That specificity is the interesting part. Not "this is better." But: *this is better precisely where the metaphor is real.*

The script that found this was not looking for physics. It was looking for mathematical structure that appears in two places where you wouldn't expect it. It found a preference matrix isomorphism and I followed it into Hamiltonian mechanics. The connection was there all along — I just needed something to notice it.

I'm writing this up as a proper paper. The method is simple enough to describe in a page. The result is clean. And the origin story — a collision between a leaderboard critique and a TypeScript chatbot that somehow produced a competitive molecular GNN — is worth documenting just to establish that this kind of search works.

The institutions I'm building are not smarter AIs. They're better ways of noticing.
